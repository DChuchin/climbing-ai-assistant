## 📋 ОСНОВНЫЕ ФАЙЛЫ

### 🤖 Для AI
- **[.cursorrules](.cursorrules)** - Базовый контекст, твой профиль
  - Автоматически читается AI в каждой сессии
  - Обновляй только при изменении базовых данных

### 📊 Данные и статистика
- **[current_state.json](current_state.json)** - Текущее состояние
  - Где ты в цикле, последние метрики
  - Обновляется AI после каждой тренировки
  
- **[progress_log.md](progress_log.md)** - Журнал тренировок
  - История всех тренировок
  - Тесты и прогресс
  - Графики показателей
  
- **[strength_metrics.json](strength_metrics.json)** - Силовые показатели
  - Детальная история всех силовых метрик
  - Автоматически обновляется AI после силовых тренировок
  - Используется для отображения прогресса на веб-странице

### 📅 План тренировок
- **[training_plan.md](training_plan.md)** - ГЛАВНЫЙ план
  - Долгосрочная стратегия (12 месяцев)
  - Текущий мезоцикл детально
  - Все 6 типов тренировок
  - Протокол тестирования
  
- **[weekly_overview.md](weekly_overview.md)** - Шпаргалка на неделю
  - Краткий обзор всех дней
  - Держи открытым на треньке
  - Визуальная структура

### 🍽️ Восстановление
- **[nutrition_recovery.md](nutrition_recovery.md)** - Питание и восстановление
  - Макронутриенты, меню
  - Добавки, гидратация
  - Сон и восстановление
  - Активный отдых

### 💪 Упражнения
- **[exercises_library.json](exercises_library.json)** - Библиотека упражнений
  - Структурированная база всех упражнений (JSON)
  - Описание, техника, видео, прогрессии
  - Используется для отображения на веб-странице
  - При добавлении новых упражнений обновляй этот файл

### 🛠️ Инструменты
- **[session_template.md](session_template.md)** - Шаблон записи тренировки
  - Скопируй и заполни после треньки
  - Или отправь AI для автозаписи

## 📊 СТРУКТУРА PROGRESS_LOG.MD

Журнал тренировок - **единый формат записи!**

### 📝 Формат записи (ОБЯЗАТЕЛЬНО):
Каждая тренировка начинается с:
```markdown
## 📅 YYYY-MM-DD | День недели | Тип тренировки

**До тренировки:**
- Сон: X часов (качество Y/10)
- Восстановление: X/10
- Состояние: [описание]
- Вес: XX кг

**Выполненная работа:**
[детали по блокам]

**Итоги:**
- RPE: X/10
- Длительность: XX мин
- Самочувствие после: X/10

**Заметки:**
- [что получилось / что сложно]

---
```

Подробнее см. **[TRAINING_LOG_FORMAT.md](TRAINING_LOG_FORMAT.md)**

### Структура файла:
```
progress_log.md
│
├── 🧪 История тестирований
│   ├── Тест #1 (базовый)
│   ├── Тест #2 (через 4 недели)
│   └── ...
│
├── 🏃 Тренировки (единый формат!)
│   ├── ## 📅 2026-02-13 | Пятница | Интенсивность
│   ├── ## 📅 2026-02-11 | Среда | Техника
│   ├── ## 📅 2026-02-10 | Вторник | Плечи
│   └── ...
│
└── Важные заметки (по необходимости)
    ├── Боли и дискомфорт
    ├── Технические прорывы
    └── Вопросы к тренеру
```

## 💬 ТИПОВЫЕ СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ

### Перед тренировкой:
1. Открой Cursor
2. Напиши: `Какая у меня тренировка сегодня?`
3. AI даст план
4. Скопируй основное на телефон или открой [weekly_overview.md](weekly_overview.md)

### После тренировки:
**Вариант А**: Через AI (быстро)
```
Закончил тренировку. Лог:
[краткие данные: самочувствие, что делал, RPE]
```

**Вариант Б**: Сам
1. Открой [session_template.md](session_template.md)
2. Скопируй шаблон
3. Заполни
4. Вставь в [progress_log.md](progress_log.md)

### Воскресенье (анализ недели):
```
Подведи итоги недели и спланируй следующую
```

### Каждые 4 недели (тест):
```
Пора на тестирование, напомни протокол
```

### Пропустил тренировку:
```
Пропустил [день], как перестроить план?
```

### Вопрос по технике:
```
Как улучшить [что-то]?
```

### Вопрос по питанию:
Сначала проверь [nutrition_recovery.md](nutrition_recovery.md), потом спроси AI если не нашел.

Просто открой Cursor и начни диалог:
